📋 WorkStation — Sequence Diagram

1. โหลดหน้า WorkStation (Page Load)

sequenceDiagram actor User participant WS as WorkStation (UI) participant API as Backend API User->>WS: เข้าหน้า WorkStation
(จาก DashboardRefurbish) WS->>WS: อ่าน orderId, operationId
จาก useParams() par โหลดข้อมูลพร้อมกัน WS->>API: GET /WorkOrderList/items_component/{orderId} API-->>WS: item_component (อะไหล่) and WS->>API: GET /WorkOrderList/workOrder/{orderId}/{operationId} API-->>WS: work (ข้อมูล Work Order) and WS->>API: GET /Mobile/GetMasterWorkorderImage?order_id={orderId} API-->>WS: masterImages (template รูป) WS->>API: GET /WorkOrderList/ImgBox/{orderId} API-->>WS: boxData (รูปที่อัปโหลดแล้ว) and WS->>API: GET /WorkOrderList/GetEmployee/{orderId}/{station} API-->>WS: getWorker (พนักงาน) and WS->>API: GET /WorkOrderList/Checked/{orderId} API-->>WS: checkedCodes (checkbox items) end WS->>WS: ตรวจ acT_START_DATE / acT_END_DATE
→ set isWorking WS-->>User: แสดงหน้า WorkStation (4 Tabs)

2. เริ่มงาน (Start Work)

sequenceDiagram actor User participant WS as WorkStation (UI) participant API as Backend API User->>WS: กดปุ่ม Start WS->>WS: เปิด Employee Modal User->>WS: เลือกพนักงาน + ยืนยัน WS->>WS: setIsWorking(true) WS->>API: POST /WorkOrderList/Employee/{orderId}/{station}
(payload: [{ORDERID, PERSONNEL_NUMBER, NAME}]) API-->>WS: สำเร็จ WS->>API: startWork() (จาก useWork Context) API-->>WS: สำเร็จ WS->>API: GET /WorkOrderList/GetEmployee/{orderId}/{station} API-->>WS: reload พนักงาน WS-->>User: แสดง Timer เริ่มนับ

3. จบงาน (Finish Work)

sequenceDiagram actor User participant WS as WorkStation (UI) participant API as Backend API User->>WS: กดปุ่ม Finish WS->>WS: Swal ถามยืนยัน User->>WS: ยืนยัน WS->>API: submitChecklist(checkedCodes)
(ส่ง checkbox items) API-->>WS: สำเร็จ WS->>API: finishWork()
(จาก useWork Context) API-->>WS: สำเร็จ WS->>WS: Swal แสดง "สำเร็จ" WS->>WS: navigate(-1) กลับหน้าเดิม WS-->>User: กลับไปหน้า Dashboard

4. เปิด Checklist (iframe)

sequenceDiagram actor User participant WS as WorkStation (UI) participant API as Backend API participant IFrame as Checklist iframe User->>WS: กดปุ่ม Check List WS->>WS: setChecklistLoading(true) WS->>API: POST /checklist/token
(payload: {orderId, station}) API-->>WS: JWT token WS->>WS: setChecklistToken(token) WS->>WS: เปิด Dialog Checklist WS->>IFrame: โหลด iframe
checklist URL + token IFrame-->>User: แสดง Checklist User->>IFrame: ทำ Checklist User->>WS: ปิด Dialog

5. อัปโหลดรูปภาพ (Upload Images - Tab 4)

sequenceDiagram actor User participant WS as WorkStation (UI) participant Service as callUploadImage participant Rename as renameImageFile participant Upload as Upload Server
(10.10.199.16:8080) participant API as Backend API User->>WS: เลือกรูป (handleFileSelect)
เก็บใน selectedFiles state User->>WS: กดบันทึกรูปทั้งหมด
(handleUploadAll) WS->>WS: Swal แสดง "กำลังอัปโหลด..." loop แต่ละรูปที่เลือก WS->>Service: callUploadImage(orderId, key, file, seq) Service->>API: GET /WorkOrderList/workOrder/{orderId}/{operationId} API-->>Service: work data Service->>Rename: renameImageFile(file, orderId, imageKey) Rename-->>Service: File ชื่อใหม่
{orderId}_{key}_{timestamp}.{ext} Service->>Service: สร้าง FormData
(file renamed) Service->>Upload: POST /upload (FormData) Upload-->>Service: imageUrl (URL รูปบน server) Service->>API: PUT /Mobile/update_workorderImage
(orderId, imageKey, imageUrl) API-->>Service: สำเร็จ end Service-->>WS: อัปโหลดเสร็จ WS->>WS: reload images (onLoad3) WS-->>User: Swal แสดง "สำเร็จ"

6. Remark / Return / QC Return

sequenceDiagram actor User participant WS as WorkStation (UI) participant API as Backend API alt กด Remark User->>WS: กดปุ่ม Remark WS->>WS: เปิด Dialog Remark User->>WS: พิมพ์ remark + ยืนยัน WS->>API: POST RemarkStop
(orderId, remark) API-->>WS: สำเร็จ else กด Return User->>WS: กดปุ่ม Return WS->>API: returnWork()
(จาก useWork Context) API-->>WS: สำเร็จ else กด QC Return User->>WS: กดปุ่ม QC Return WS->>API: qcReturnWork()
(จาก useWork Context) API-->>WS: สำเร็จ end WS->>WS: navigate(-1) / reload WS-->>User: กลับหน้าเดิม

7. แก้ไข / ลบ อะไหล่ (Tab 3)

sequenceDiagram actor User participant WS as WorkStation (UI) participant API as Backend API alt แก้ไขจำนวน User->>WS: กดปุ่ม Edit (แก้ qty) WS->>WS: เปิด Dialog Edit QTY User->>WS: ใส่จำนวนใหม่ + ยืนยัน WS->>API: POST /Mobile/SetWorkOrderSparePart
?OrderId={orderId}

(จาก useWork Context) API-->>WS: สำเร็จ WS->>API: GET /WorkOrderList/items_component/{orderId} API-->>WS: reload อะไหล่ end WS-->>User: แสดงรายการอะไหล่ใหม่
